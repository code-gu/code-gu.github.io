Git常用命令学习

通过上一文，我们明白了Git的一大功能点：版本控制，接下来我们将学习如何使用Git

Git的使用方式

Git 有多种使用方式。 你可以使用原生的命令行模式，也可以使用 GUI 模式，这些 GUI 软件也能提供多种功能。本文中，我们将使用命令行模式。 这是因为首先，只有在命令行模式下你才能执行 Git 的 所有 命令，而大多数的 GUI 软件只实现了 Git 所有功能的一个子集以降低操作难度。 如果你在命令行模式下能操作的得心应手，那么GUI下几乎没有上手难度，反之则不成立。

个人还是推荐使用命令行模式，程序员应当具有钻研精神，才能在这条路上走的更远

在你安装完成Git后，我们创建Git仓库，随便在桌面或者文件夹内右键可以看到Git Bash Here的操作，我们在桌面下点击，并输入命令pwd

可以看到，显示的是当前目录，也就是你想在哪里创建Git进行项目管理，就在该文件目录进行Git Bash Here，这里我默认在D盘的git_test文件夹下==创建Git仓库==

三种区域

首先是三种区域，简单地说就是Git存在三个区域，彼此配合工作完成本地的版本控制
工作区、暂存区以及 Git 目录(历史记录区)

三种文件状态

Git 有三种状态，你的文件可能处于其中之一： 已提交（committed）、已修改（modified） 和 已暂存（staged）

版本回退

通过上一文，我们得知张伟通过版本回退，逆转了局势，完美的和女神完成了第一次约会，那么他的操作是如何实现的呢，有两种方法
首先我们查看有版本库有几个版本git log

简单讲解一下commit后面跟的一大串字符，这是git通过hash算法的sha-1加密算法，这种算法，通常用于检验文件完整性，在这里我们称为版本id

第一种方法git reset –hard head^

我们回到了第一次的提交版本,这里说明一下

head后面的^代表回退一个版本，几个^代表回退几个版本

我们再查看版本

第二种方法git reset –hard

我们通过git log查看版本可以看到多个版本，其中commit后面一大串是版本id,而我们可以通过版本id进行版本回滚

注意：我们只需要使用hash串的前面几个字符即可，因为多个文件的sha-1加密后不会产生唯一的hash值，但是建议使用4个以上的字符代替版本号

问题

可以看到我们第二次提交的信息也没有了，但是问题也来了==如果出现了误操作，导致回退了不想用的版本，如何解决？== git的又一个强大之处出现了：git总有后悔药给你吃
我们通过命令git reflog可以查看最近的一些操作记录

可以看到之前对应操作过后的版本id
然后我们通过第二种方式git reset --hard 版本id回退到误操作之前的版本，再继续你想要的操作即可

总结

  1. git add 文件名orgit add .推送一个或者全部文件到暂存区
  2. git commit把暂存区文件提交为一个新的版本
  3. git reset --hard head^回退上一个或多个版本
  4. git reset --hard 版本id回退某个指定的版本
  5. git status查看文件状态
  6. git reflog查看最近操作
